第一章 介绍

1.1 关于APB协议

  该APB是一种低速、低功耗、简单的接口协议。也是一种非流水线的同步接口协议，每次传输耗费两个周期。专门设计用于访问外设的寄存器，外设通过apb桥与内存建立连接以实现这一操作。例如：AXI2APB桥能够将多个apb外设与一个axi进行连接。

  一次APB传输应当由APB桥发起，因此APB桥也可被称为传输的请求方（Requester）；由外设对请求作出响应，因此外设可被称为响应方（Completer）。

  本规范将使用Requester和Completer表示传输的发起者(主机)和执行者(从机)。

第二章 信号描述

2.1 AMBA APB 信号

  • 有些信号的位宽是可变的；
  • 有些信号的位宽属固定的；
APB版本信号名含义位宽信号源描述
Apb2PCLK总线时钟1CGU同步所有触发器的输入信号，且上升沿采样
PRESETn总线复位1RGU复位所有触发器的输出信号，n表示低有效
PSELx从机片选1主机(译码器)每个主机都应该有一个PSELx信号，表明当前从机是否被选中；
PENABLE从机使能1主机在操作的第二个周期拉高，表示开始进行数据读写
PWRITE读写使能1主机在WRITE为0时，读使能，在WRITE为1时，写使能
PADDR读写地址ADDR_WIDTH主机对读写的从机和寄存器进行选择
PWDATA写数据DATA_WIDTH主机要写入到从机的数据
PRDATA读数据DATA_WIDTH从机要读出到到主机的数据
Apb3PREADY传输等待1从机在数据接收方未准备好时，由从机发出，延长主机发起的传输
PSLVERR错误响应1从机由从机发出，表明此次传输了一笔错误的数据
Apb4PPROT事务类型3主机表明事务的访问类型
PPROT[0]:正常访问(normal{0})还是特权访问(privileged{1})
PPROT[1]:安全访问(secure{0})还是非安全访问(non-sceure{1})
PPROT[2]:数据访问(data{0})还是指令访问(instruction{1})
PSTRB写更新DATA_WIDTH/8主机表明写传输要更新的字节
Apb5PNSE保护类型1主机RME系统架构，PSEL被断言时，PNSE必须有效
PNSEPPROT[1]物理地址空间
00Secure
01None-Secure
10Root
11Realm
PWAKEUP唤醒从机1主机可以在psel被断言前，中，后任一过程
PAUSERuser请求属性USER_DATA_WIDTH主机建议最大位宽为128bit
PWUSERuser写数据USER_DATA_WIDTH主机建议最大位宽为DATA_WIDTH/2
PRUSERuser读数据属性USER_DATA_WIDTH从机建议最大位宽为DATA_WIDTH/2
PBUSERuser响应属性USER_DATA_WIDTH从机建议最大位宽为16bits

Note1: PPROT主要用于安全事务和非安全事务的标识符来使用，PPROT[0]和PPROT[1]可以为其赋予不同的解释。

Note2: PSTRB不应该在读操作时被拉高。

2.1.1 地址总线

 地址总线只有一条，且以byte为单位，用于写传输和读传输；

 地址总线可以和数据位宽不对齐，但结果是不可预测的；

2.1.2 数据总线

 数据总线有两个，互相独立，一条为写数据总线，一条为读数据总线。

 数据总线的单位是Byte，且读写数据总线的位宽必须一致。

 不可以同时进行读写传输，因为他们没有各自的握手信号。

第三章 传输

3.1 写传输

  • 包含两种写传输方式
    • 有等待写；
    • 无等待写；

3.1.1 无等待写

时序图wavedrom源码[折叠代码块]
 
{signal: 
[
   {name:'PCLK', wave: 'P...',period:1},
   {name:'paddr', wave: '==.=',data:["","Addr1",""]},
   {name:'PWRITE', wave: '=1.=',data:["",""]},
   {name:'PSEL', wave: '=1.=',data:["",""]},
   {name:'PENABLE', wave: '=01=',data:["",""]},
   {name:'PWDATA', wave: '==.=',data:["","Data1",""]},
   {name:'PREADY', wave: '=.1=',data:["",""]},
 ],
   head:
   {
   tick:0,
   every:1
 },
   foot:
   {
   text:"图3.1.1 无等待的写传输"
   } 
}
  • 第一个周期[setup]，PSEL拉高，且PADDR，PWRITE，PWDATA必须有效；
  • 第二个周期[access]，PENABLE拉高，PREADY被从机拉高表明接受一笔传输，且PADDR,PWRITE,PWDATA保持不变；
  • 第三个周期[idle]，PENABLE和PSEL拉低。
