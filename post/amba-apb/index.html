第一章 介绍

1.1 关于APB协议

  该APB是一种低速、低功耗、简单的接口协议。也是一种非流水线的同步接口协议，每次事务耗费两个周期。专门设计用于访问外设的寄存器，外设通过apb桥与内存建立连接以实现这一操作。例如：AXI2APB桥能够将多个apb外设与一个axi进行连接。

  一次APB事务应当由APB桥发起，因此APB桥也可被称为事务的请求方（Requester）；由外设对请求作出响应，因此外设可被称为响应方（Completer）。

  本规范将使用Requester和Completer表示事务的发起者(主机)和执行者(从机)。

2.1.1 地址总线

 地址总线只有一条，且以byte为单位，用于写事务和读事务；

 地址总线可以和数据位宽不对齐，但结果是不可预测的；

2.1.2 数据总线

 数据总线有两个，互相独立，一条为写数据总线，一条为读数据总线。

 数据总线的单位是Byte，且读写数据总线的位宽必须一致。

 不可以同时进行读写事务，因为他们没有各自的握手信号。

第三章 事务

3.1 写事务

  • 包含两种写事务方式
    • 有等待写；
    • 无等待写；

3.1.1 无等待写

   {name:'PSEL', wave: '=1.=',data:["",""]},
   {name:'PENABLE', wave: '=01=',data:["",""]},
   {name:'PWDATA', wave: '==.=',data:["","Data1",""]},

 ],

   {
   tick:0,
   every:1
 },
   foot:

   text:"图3.1.1 无等待的写事务"
   } 
}
  • 第一个周期[setup]，psel拉高，且paddr、pwrite、pwdata必须有效；
  • 第二个周期[access]，penable拉高，pready被从机拉高表明接受一笔事务，且paddr、pwrite、pwdata保持不变；
  • 第三个周期[idle]，penable和psel拉低。

3.2 write strobe（写选通）

  • PSTRB信号为数据事务提供了一种稀疏的方式，也就是将一笔数据分开几次事务。
  • 每次PSTRB事务对应了1byte的数据。
  • PSTRB为高表明当前属于数据事务的一部分即1byte数据有效；
31  2423  1615  87  0
PSTRB[3]PSTRB[2]PSTRB[1]PSTRB[0]
  • 对于读事务来说，主机发送的PSTRB所有byte必须拉低。

3.2.1 PSTRB的兼容性

  • 由于PSTRB属于可选的信号，因此主机与从机是否有该信号会引起兼容性的问题，其兼容性应遵循下面的列表。
PSTRB从机不存在该信号从机存在该信号
主机不存在该信号兼容，不支持稀疏写兼容，支持稀疏写，PSTRB应该与PWRITE直接连接
主机存在该信号兼容，不支持稀疏写兼容，不支持稀疏写

3.4 错误响应

  • PSEVERR可以用来表示事务是否有误，这在写和读的过程中都会发生。
  • PSLVERR在事务的最后一个周期有效，即PENABLE、PSEL、PREADY都为高的周期。
  • 有个建议但并不强制要求，在PENABLE、PSEL、PREADY为低的周期，PSLVERR也为低。
  • 事务中包含错误是否改变的外设的状态，这取决于外设如何处理。
  • 当写事务接收到一个错误，并不意味着外设的寄存器没有被更新。
  • 读取事务在接收到错误时可能会返回一个无效的数据。对于读取错误，不要求外设将数据总线驱动到0，外设可能仍会使用错误数据。从机不能够根据错误响应来阻止读取PRDATA上的值。
  • 从机不要求支持PSLVERR，如果从机不包含PSLVERR，主机的PSLVERR输入应当绑定为LOW。

3.5 保护机制

  • 为了支持复杂的系统设计，经常需要为系统中的设备提供防止非法事务的保护。
PPROT保护类型描述信号值和对应保护类型
PPROT[0]Normal或PriviledgedPriviledged在系统访问中处于最高优先级LOW：normal
HIGH：privileged
PPROT[1]Secure或Non-secure该位用于处理需要更大处理模式的系统LOW：secure
GIGH：None-secure
PPROT[2]Data或Instruction该位用于提示作用LOW：data
HIGH：Inst

Note：PPROT最主要的作用就是作为Secure或者Non-secure的提示符

3.5.1 PPROT的兼容性

  • PPROT对于主机和从机都是可选的。
PPROT从机不存在该信号从机存在该信号
主机不存在该信号兼容不兼容，如果在功能上保证了正确就是兼容的不明白
主机存在该信号兼容，从机无法访问该信号，因此该信号可以被忽略兼容
